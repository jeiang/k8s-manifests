# rbac-access

Helm chart for cluster and namespace RBAC.

## What This Chart Creates

- `aidan` as a cluster admin.
- `saeed` as namespace admin in `shared`, `saeed`, and declared namespaces prefixed with `saeed-`.
- `noel` as namespace admin in `shared`, `noel`, and declared namespaces prefixed with `noel-`.
- `gilliano` as namespace admin in `shared`, `gilliano`, and declared namespaces prefixed with `gilliano-`.
- The namespaces listed above.

## Dependencies

- Helm 3 and `kubectl`.
- The installer must have permission to create `Namespace`, `RoleBinding`, and `ClusterRoleBinding` resources.
- The built-in `cluster-admin` and `admin` ClusterRoles must exist.
- Values under `admin.subjects` and `namespaceAccess.users[].subjects` must match identities recognized by cluster authentication.

## Install

```fish
helm lint ./rbac-access
helm template rbac-access ./rbac-access --namespace kube-system
helm upgrade --install rbac-access ./rbac-access \
  --namespace kube-system
```

## Verify

```fish
kubectl get clusterrolebinding rbac-access-admin
kubectl get namespaces shared saeed noel gilliano
kubectl get rolebinding --all-namespaces -l app.kubernetes.io/instance=rbac-access
```

## Values

See [`VALUES.md`](./VALUES.md) for the local values documented with defaults and operational notes.

If the namespaces already exist and are not managed by this chart, set:

```yaml
namespaces:
  create: false
```

Kubernetes RBAC only authorizes identities that your cluster authentication already recognizes. Update `admin.subjects` and each `namespaceAccess.users[].subjects` entry to match exact usernames, groups, or service accounts.

For namespace users, `namespaceAccess.clusterRole` defaults to the built-in `admin` ClusterRole. That grants broad permissions inside the listed namespaces only; it does not grant cluster-wide access.

Kubernetes RBAC does not support namespace wildcard bindings. Prefix access is generated by this chart against the namespaces declared in values. To grant access to a future namespace such as `saeed-website`, add it to `namespaces.additional` and run `helm upgrade`:

```yaml
namespaces:
  additional:
    - saeed-website
```

To add another namespace-scoped user, add a new entry under `namespaceAccess.users`.
